# 면접 준비 자가 구축 — D-3 질주의 기록

## 한 줄

특정 회사의 대규모 공개 채용 마감 3일 전에 시작해, 이력서·테크 딥다이브·면접 학습용 웹 패키지를 자가 구축한 프로젝트. 결과적으로 "이력서 한 장"을 위한 도구가 별도 미니 프로덕트로 자라난 기록.

## 컨텍스트 / 동기

어떤 채용 사이클은 너무 빨라서, 발견한 순간 이미 D-3이다. 이번도 그랬다. 같은 사이클을 놓치면 다음 대규모 공고까지 또 기다려야 하는 종류의 공고였고, 한 번 해보기로 했다.

본 프로젝트의 실제 동기는 두 갈래였다.

1. **경력의 재해석.** 현 직무에서 쌓은 미션크리티컬 레거시 경험을, 지원 대상 회사의 기준(분산 트랜잭션, 메시지 큐 전환, 관측 시스템 구축 등)으로 다시 읽어내는 작업. 같은 사실이 "레거시 운영자"로 압축되느냐, "시스템 현대화 경험자"로 재프레이밍되느냐가 관건이었다.
2. **학습 도구의 자가 구축.** 면접 준비 자료를 마크다운 더미로 두지 않고, React + TypeScript + Vite로 자체 학습 웹앱을 만들어 돌리면서 공부하는 실험.

## 핵심 결정

### 1) 이력서는 "상황 · 해결 · 성과" 3단 통일

스펙 단계에서 성과 7개를 뽑고 각각을 "상황·해결·성과" 3단 구조로 강제 통일했다. 도메인 세부는 빠지고 문제 해결 구조만 남게. 6개는 본문 코어로, 1개는 참고로 분류.

### 2) 동기는 "성장"이 아니라 "문제 해결 경험의 이전"

지원 동기 초안에서 "성장하고 싶습니다" 류 표현을 전부 제거하고, "레거시 환경에서 분산 트랜잭션·메시지 큐 전환·시스템 현대화를 직접 해왔다 → 이 경험을 현대적 스택 위에서 더 큰 규모에 적용하고 싶다"로 통일. 미션크리티컬 경험과 금융 트랜잭션 정확성·안정성을 본질이 같은 문제로 연결.

### 3) 테크 딥다이브를 별도 트랙으로

이력서 본문은 압축돼야 하지만 면접에서 풀어낼 살이 필요하다. 6개 토픽 폴더(`01-distributed-tx`, `02-rabbitmq`, `03-elasticsearch`, `04-mcp-llm`, `05-ejb-to-spring`, `06-pi-sap`)에 각각 README + questions.md(난이도 5단계 + 빈도 표시) + study/ 학습 가이드를 만들었다. 면접 질문은 토픽별 16~30개, **총 139개**.

### 4) 면접 학습용 웹앱 자가 구축

질문 139개를 그냥 마크다운으로 두지 않고 React + Vite + TypeScript로 자가 학습 도구를 만들었다. 대시보드 → 프로필별 라우팅(`/:profileId/{resume,study,quiz,interview}`), Resume 탭, Study 탭(주제·서브주제 라우팅), Quiz 탭(난이도·빈도 필터·힌트), Interview 탭, 지원 회사 사이트 분석 기반 용어사전(glossary).

결과적으로 "이력서 한 장"을 위한 도구가 별도 미니 프로덕트가 됐다. 의도된 비대화는 아니었지만 학습 효과는 일부 있었다.

### 5) 지원 회사 사이트 직접 탐색

Playwright MCP로 지원 회사 사이트를 직접 돌며 용어사전 페이지·검색 동작·다크 테마를 캡처. 스크린샷 11장이 `.playwright-mcp/`에 남았다. 지원 회사의 용어사전 검색 동작을 본인 학습 도구의 `Search` 컴포넌트 설계 레퍼런스로 차용.

## 스택과 구조

- **이력서·문서 형식**: 마크다운. `docs/specs/`(Q&A 정리) → `docs/research/`(회사 리서치) → `resume/draft-v1.md` 단방향. `PLAN.md`로 Phase 1~6 진행 상황 추적.
- **이력서 본문 톤**: 격식체 + 명사형 압축. "상황 → 해결 → 성과" 3단 강제.
- **테크 딥다이브 구조**: 토픽 폴더 6개, 각각 README + questions.md + study/. 질문 메타에 5단계 난이도(⭐~⭐⭐⭐⭐⭐), 빈도(🔥/🟡/🔵).
- **학습 웹앱**: React 19 + TypeScript + Vite + React Router + pnpm. 컴포넌트 트리 `Layout / Dashboard / Resume / Study / Quiz / Interview / Search / ThemeSelector`. 데이터는 TS 모듈(`quiz.ts` 120KB, `glossary.ts` 69KB, `interview.ts` 28KB). 멀티 프로필 지원(여러 이력서 버전 동시 운용).
- **회사 사이트 분석**: Playwright MCP로 직접 탐색·스크린샷 캡처. 다크 테마·글래스모피즘 스타일을 자체 학습 도구에 일부 차용.

## 결과·현황

- **Phase 1 (Spec) 완료** — career-profile / tech-stack / achievements(7건) / motivation 4문서 확정.
- **Phase 2 (리서치) 부분 완료** — 포지션·이력서 가이드 작성. 기업문화 리서치는 미작성으로 남음.
- **Phase 3 (이력서) draft-v1까지** — 2026-03-30 23:50 최종 수정. final.md는 만들지 않았다. `PLAN.md` 상태가 "이력서 리뷰 중"에서 멈춰 있다.
- **Phase 4~6 (코테·면접) 미진행** — 테크 딥다이브 학습 자료(139개 질문)와 웹앱은 만들었지만 실제 풀이·모의 면접은 세션에 흔적이 없다.
- **마지막 활동**: 2026-03-31 00:23. 마감일 새벽까지 작업.

## 회고

**잘된 것.**

- 경력 사실을 지원 회사 기준으로 한 번에 재해석한 골격이 단단하다. 같은 사실을 어떤 회사에 가도 다시 쓸 수 있는 자산이 됐다.
- 면접 질문 139개를 토픽·난이도·빈도로 분류한 인덱스. 다른 회사 지원에도 90%는 재사용 가능.
- "성장하고 싶다" 류 동기를 전부 도려내고 "문제 해결 경험의 이전"으로 톤을 통일한 것.

**약한 부분.**

- 학습 웹앱이 본질에서 벗어났다. 마감 D-3에 React+TypeScript 멀티 프로필 라우팅을 짜고 있을 시간에 코테 1문제라도 풀었어야 했다. **도구 만족이 학습을 대체하는 함정.**
- 현대적 스택 실무 부족을 매칭표에서 인지했지만 그걸 메우는 액션으로 이어가지 못했다. 인지로 끝났다.
- `final.md` 미작성. draft-v1에서 멈췄다는 건 마감 임박에서 자기 검열이 작동했거나 우선순위를 다른 데로 옮겼다는 신호.
- Phase 4~6(코테·면접) 미진행. "준비"의 80%가 "준비를 위한 자료 만들기"에서 끝났다.

**다시 한다면.**

- **D-3에 시작하지 말 것. 최소 D-30.**
- 학습 도구는 만들지 말 것. 마크다운 + 종이로 충분. 도구 만들고 싶은 충동이 오면 그게 회피 신호.
- 코테는 첫날부터 매일 1문제 강제. 자료는 1주차에 마감하고 2~3주차는 풀이·모의 면접.
- 이력서 final은 완벽 추구로 미루지 말고 "draft-v2 = final" 룰로 강제 마감.
